# AGENTS.md — Kickoff

Match-day arrival router + watch-party guide for the FIFA World Cup in Miami.

## Prime directive — NEVER hallucinate a departure time
Departure times, shuttle windows, and routes are COMPUTED in code from seeded data, never
generated by an LLM. The model (translation/voice) only does speech-to-text + translation +
filling a fixed template. If a fact isn't in seeded data with a source, the app doesn't state it.

## Tiered scope (always keep a working demo)
- MUST SHIP: Schedule + Arrival & Transport.
- HIGH PRIORITY: Heat map (watch parties + restaurants).
- DELIGHT (cut first): Translation, Music.

## Stack
- Next.js App Router + TypeScript + Tailwind + shadcn/ui. Vercel deploy.
- Map: react-leaflet or Google Maps JS. Data: Google Places (primary), Spotify embed, a
  translation + STT API. Yelp only if a key already works (access is approval-gated).
- Tabs: Schedule | Plan | Map | Voice. All keys server-side only — never NEXT_PUBLIC_ a secret.

## Non-negotiables
- Times go through `lib/arrivalEngine.ts` (deterministic). No free-text time generation anywhere.
- Every fact-bearing line shows a source from `data/sources.json` (tap to open).
- Cache/seed all external data — NEVER demo a live scrape or API call on venue wifi.
- Label estimates "estimate" and modeled data "modeled." Be honest in copy.

## Definition of done
Pick match + origin → ranked plan with leave-by + sources renders; drive shows the parking
hard stop; map shows seeded watch-party pins; one Spanish/Portuguese voice query reads back
a correct, template-filled plan.
